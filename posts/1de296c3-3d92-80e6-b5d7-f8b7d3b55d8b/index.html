동영상 최적화

동영상 최적화에 있어서 제일 중요한 건 동영상의 용량을 줄이는 것

  1. 동영상 압축

    https://www.media.io/

    화질이 저하되므로 상황에 따라 적절하게 조절하는 것이 좋음

  2. 동영상 길이 줄이기

  3. 적절한 동영상 포맷 사용(mp4 → webm)

  4. CDN 서비스 이용

    캐시를 이용해 효율적으로 리소스를 다룰 수 있음

이미지 최적화

  1. Lighthouse

    Google에서 개발한 웹 페이지의 품질을 평가하기 위한 자동화 도구

    웹페이지의 문제점을 찾고, 개선방안을 제안받을 수 있음

  2. 이미지 압축

    https://squoosh.app

    WebP : 구글에서 만든 이미지 포맷, 이미지 품질이 jpg, png와 비슷할 때는 다른 형식 대비 파일 크기(용량)이 작음

  3. CDN 서비스

    https://cloudinary.com

    클라우드 기반의 이미지 및 비디오 관리 서비스(업로드 및 공유 가능)

    이미지의 크기나 퀄리티를 url로 손쉽게 조절 가능

    • c_fill(width, height 내에 이미지가 깨지지 않고 들어가도록 맞춤)
    • q_auto:best(최적화)

폰트 최적화

FOIT와 FOUT

  • FOIT

    flash of invisible text

    폰트를 다운로드 하기 전에는 텍스트를 노출하지 않음

  • FOUT

    flash of unstyled text

    폰트가 다운로드 되기 전에는 기본 폰트를 노출

    다운로드 후에 폰트 교체

Font Display

  • swap(FOUT): 폰트를 다운 받기 전에는 기본 폰트 노출, 다운로드 완료 후 폰트 교체
  • block(FOIT): 3초 내에 폰트를 다운받지 못하면 기본 폰트 노출
  • fallback(FOIT): 0.1초 정도 block이 발생, 3초 이내로 다운받지 못하면 다운로드 여부와 상관 없이 기본 폰트 노출(캐시)
  • optional(FOIT): fallback과 유사, 폰트가 다운로드 받는 시간이 너무 오래 걸리면 브라우저가 연결 취소 가능(캐시)

  1. WOFF2

    EOT → TTF/OTP → WOFF → WOFF2 순으로 용량이 작아짐

    단, WOFF2는 지원 범위가 좁다

  2. Subset

    필요한 글자들만 추려서 폰트를 생성하는 것

    https://namu.wiki/w/완성형/한글 목록/KS X 1001

    https://transfonter.org/

  3. 폰트 로드 속도 개선(Preload)

    webpack-font-preload-plugin - npm (npmjs.com)

    craco.config.js에 추가

    const CracoAlias = require('craco-alias')
const FontPreloadPlugin = require('webpack-font-preload-plugin')

module.exports = {
  plugins: [
    {
      plugin: CracoAlias,
      options: {
        source: 'tsconfig',
        tsConfigPath: './tsconfig.paths.json',
      },
    },
  ],
  webpack: {
    plugins: {
      add: [
        new FontPreloadPlugin({
          extensions: ['woff2'],
        }),
      ],
    },
  },
}

렌더링 최적화

memo: Props가 변경되지 않는 한 리렌더링되지 않는다

→ Props가 빈번하게 바뀌는 컴포넌트는 memo를 쓰지 않는 것이 좋다

useCallback: 함수 정의를 캐싱함(리렌더링 시 함수를 새롭게 만들지 않겠다)

로직 추상화

렌더링에 집중할 수 있는 컴포넌트 환경 == 비즈니스 로직에 대한 코드와 화면을 그리는 컴포넌트 코드가 역할에 맞게 명확하게 분리되어 있는 환경

→ 재사용성, 가독성 및 유지보수 측면, 테스트 측면에서 유리함

선언적인 코드란 무엇인가

동작이 예상 가능한 추상화된 코드

React Query

서버의 상태를 관리하는 라이브러리

선언적으로 상태와 에러, 로딩 관리 가능

캐싱, 값 업데이트 등 비동기 과정을 편하게 사용할 수 있도록 도와줌

에러 처리

  1. 안정성

    하나의 컴포넌트가 고장나더라도 전체 애플리케이션에 영향을 미침

  2. 유저 경험

    에러 발생 시, 유저에게 유의미한 정보 전달 가능

  3. 로그

    에러 정보와 함께 디버깅 정보 로그 시스템에 보내 문제를 인지하고 해결할 수 있도록 함

ErrorBoundary

하위 컴포넌트에서 발생하는 JavaScript 관련 에러를 감지하고 fallback UI를 노출하거나 공통 에러 로깅을 도와주는 클래스 컴포넌트

에러가 발생한 특정 부분만 격리시키고 나머지 부분은 정상적으로 작동하게 할 수 있음
