디자인 시스템

재사용 가능한 컴포넌트와 패턴을 사용하여 규모있는 디자인을 하는 규칙과 패턴들의 모음

재사용 가능한 컴포넌트와 가이드이며, 인터페이스와 그 상호작용을 만들 때 활용

재사용 가능한 컴포넌트의 모음으로 일관된 경험을 제공하고 다양한 프로그램을 구축할 수 있는 확장성을 갖고 있음

⇒ 재사용 가능한 컴포넌트, 패턴 그리고 가이드/ 인터페이스를 개발할 때 사용되고, 일관된 경험을 사용자에게 제공하며, 다양한 프로그램으로 확장 가능

  • Material Design
  • Shopify Polaris
  • Carbon Design System
  • Open Design System

구성요소

  • 디자인 원칙/철학

    비전, 미션, 가치 설정하기

  • 스타일가이드

    로고, 칼라, 레이아웃/그리드 등

  • 컴포넌트 라이브러리

    재사용이 가능한 다양한 UI 기본구성 요소

  • 패턴 라이브러리

    자주 반복되어 사용되어지는 디자인의 템플릿(컴포넌트의 조합)

  • 문서화

  • 시스템 관리 운영

    사내 공유되는 디자인 시스템을 효과적으로 관리하고 유지하기 위한 프로세스와 규칙

디자인 시스템의 장점

  • 디자인 일관성 유지
  • 브랜드 강화
  • 효율적인 개발
  • 시간 단축
  • 팀 간 협업 강화
  • 빠른 온보딩
  • 유지 보수 용이
  • 높은 품질의 경험

재사용성과 유지보수성

  • 재사용성

    한 번 작성된 코드가 다양한 환경이나 프로젝트에서 재사용될 수 있는 능력

    한 번 작성된 공통 컴포넌트나 스타일, 함수 등이 여러 페이지나 프로젝트에서 재사용될 수 있는 능력

  • 유지보수성

    코드나 시스템의 변경, 수정, 개선이 쉽고 효율적으로 이루어지는 정도

    서비스의 기능, 화면에 새로운 요구사항을 반영하거나 수정을 쉽고 효율적으로 이루어지는 정도

재사용성을 위해 할 수 있는 것들

  1. 컴포넌트화
  2. 공통 라이브러리 및 유틸리티 사용
  3. 디자인 시스템 구축
  4. API 추상화, 코드 디자인 등

유지보수성을 위해 할 수 있는 것들

  1. 코드의 가독성과 일관성 향상
  2. 테스트 자동화
  3. 문서화
  4. 모듈화와 코드 분리, 응집도 등
  5. 리팩터링(기술 부채)

소프트웨어 개발 사이클

소프트웨어 개발(서비스, 기능, 리팩터링)

  1. 왜 이 라이브러리를 쓰지? 왜 이 기술을 사용하지?
  2. 왜 이것을 해야하지? 어떤 목적을 가지고 있지? 누가 쓰는 거지?

디자인 시스템 vs. UI 라이브러리

  • 디자인 시스템

    브랜드와 제품의 일관성을 유지하고, 팀 간의 협업을 향상시켜 더 빠르고 효과적으로 UI를 디자인하고 구축하는데 도움이 됨

    사용자 인터페이스를 구축하는데 필요한 모든 요소를 정의하는 포괄적인 문서와 가이드라인 집합(컴포넌트, 패턴, 디자인 원칙, 사용 가이드라인, 툴, 스타일 가이드 등)

  • UI 라이브러리

    코드의 재사용성을 향상시키고, 팀 간의 협업을 간소화하여 제품의 UI를 빠르게 구축하는데 도움이 됨

    구체적인 코드를 포함하는, 재사용 가능한 UI 컴포넌트들의 모음(실제 개발에서 사용 가능한 버튼, 입력 폼, 카드 등의 구현체)

디자인 시스템에 기대하는 것

  1. 형태

    색상 팔레트, 타이포그래피, 컴포넌트

    • 제약
    • 자유

  2. 기능

    입력(클릭, 타이핑 등), 선택(라디오, 체크박스, 토글 등)

  3. 접근성

    컬러 대비, 키보드 네비게이션, aria, 포커스

디자인 시스템의 구조

  1. 원칙

    디자인과 개발에 대한 핵심 가치 및 지침

  2. 테마

    색상, 타이포그래피, 그리드, 스페이싱 등과 같은 기본 디자인 요소에 대한 지침

  3. 컴포넌트

    버튼, 입력란, 탭, 모달 등과 같은 재사용 가능한 UI 요소의 라이브러리

  4. 패턴

    여러 컴포넌트를 결합하여 만들어진 복잡한 사용자 인터페이스 요소

  5. 도구 및 유틸리티

    디자인 시스템을 실제 제품에 통합하기 위한 도구와 플러그인

  6. 문서화

    디자인 시스템의 모든 요소를 사용하는 방법 등

  7. 가이드라인

    좋은 사용성, 접근성 등의 가이드와 권장사항들

  8. 프로세스 및 워크플로우

    디자인 시스템 업데이트, 확장하는 것에 대한 프로세스

디자인 토큰

bookmark

사용자 인터페이스를 구축하는 방법에 대한 자세한 정보를 전달하기 위한 디자인과 개발 간의 공유 언어

간격, 색상, 타이포그래피, 개체 스타일, 애니메이션 등과 같이 디자인 시스템을 구성하고 유지하는데 필요한 값으로 구성

가능한 모든 시나리오에 대해 focused set을 갖는 것

  • Raw Value

    원시값/ 본래의 값

  • Global(Primitive/ Core/ Base/ Foundation/ Root)

    글로벌/ 사용 맥락에 상관 없이 디자인 언어의 기본값

  • Alias(Semantic/ Applied/ Purpose)

    별칭/ 특정 사용 맥락과 의도를 전달할 때 쓰이는 값

  • Component(Overwrites/ Scoped)

    컴포넌트/ 컴포넌트와 관련한 모든 디자인 속성을 가진 값

Adobe Spectrum의 경우 네이밍을 3파트로 나누어 구성한다

  • context

    가장 광범위한 아이디어

  • common unit

    가장 일관된 아이디어

  • clarification

    가장 구체적인 아이디어

웹, 네이티브 등을 포함한 여러 플랫폼과 디바이스에 걸쳐 디자인을 확장하기 위한 기술에 구애받지 않는 아키텍처 및 프로세스

하드 코딩된 값과 변수 대신 사용되어 모든 제품 경험 또는 플랫폼에 걸쳐 유연성과 통합성을 보장

플랫폼에 구애받지 않기 때문에 개발자와 디자이너 간 커뮤니케이션이 수월해진다

  • Values

    색상 값, 절대 및 상대 길이 단위, 백분율, 숫자 등과 같은 코드의 기본 값

  • Base Variables

    컨텍스트에 구애받지 않는 이름으로 표시되는 디자인 언어의 기본 값

    직접 사용할 수 있으며 다른 모든 토큰 유형에 상속됨

  • Common Tokens

    특정 컨텍스트 또는 추상적 개념과 관련 있음

    토큰의 의도된 목적을 전달하는데 도움이 되며 단일 의도를 가진 값이 여러 위치에 나타날 때 효과적

  • Component Tokens

    이를 사용하는 특정 컴포넌트와 관련 있음

    대상 속성 뿐만 아니라 해당 상태도 전달함

토큰 이름은 기본값을 보지 않고도 이 토큰을 사용해야 하는 시기와 위치를 전달하는 엄격한 명명 규칙을 따름

배리어블 규칙

  1. Namespace

    1. System

      다른 디자인 시스템과 구분하기 위해 주로 모든 배리어블/토큰 앞에 접두어로 시작

      | Material | —md- | | --------------- | ----------- | | Adobe Spectrum | spectrum- | | Shopify Polaris | —p- | | Line | —lsdg- |

  2. Object

    1. Component

      디자인 시스템 내에서 특정 컴포넌트의 스타일 및 레이아웃을 적용하고 싶을 때 사용

      | 버튼 | —p-button-primary-bg-color | | -- | ---------------------------- | | 카드 | —p-card-bg-color | | 헤더 | —p-header-bg-color | | 인풋 | —p-input-bg-color |

  3. Base

    1. Category

      UI 기본 구성 요소를 공통된 유형으로 그룹화

      • Color, Font, Sizing, Spacing, Radius, Opacity, Elevation, Breakpoints, Shadow, Duration

    2. Property

      각 카테고리 내에서 특성에 따라 세분화

      | Color | brand background border text text link | color-brand- color-background- color-border- color-text- color-textLink- | | ------ | -------------------------------------------- | ---------------------------------------------------------------------------------- | | Font | style size weight letter spacing line height | font-style- font-size- font-weight- font-letterSpacing- font-lineHeight- | | Sizing | border square width | sizing-border- sizing-square- sizing-width- |

  4. Modifier

    하나의 디자인 요소를 여러 상황, 상태에 따라 변형할 수 있도록 한다

    1. Variant
      • Hierarchy(시각적 위계질서)
        • primary, secondary, tertiary
      • Feedback(기능)
        • success, error, information, warning
    2. State
      • Interactive states(인터랙션 상태)
        • default, hover, press/active, focus, disabled, visited, error
    3. Scale
      • 숫자
        • 1, 2, 3, 4
      • 티셔츠 사이즈
        • xs, sm, md, lg, xl
      • 강도
        • low, normal, high, max
      • 세기
        • weak, moderate, strong, extreme
      • 스피드
        • instant, immediate, quick, fast, moderate, slow

⇒ —p-color-bg-interactive-subdued-hover

⇒ $color-background-primary-strongest
