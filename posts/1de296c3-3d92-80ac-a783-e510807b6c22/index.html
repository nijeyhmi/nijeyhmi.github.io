2025. 04. 23.
React

함수형 프로그래밍

프로그램을 함수의 계산으로 본다

  • 함수가 1급 객체/1급 시민
  • 불변성(Immutability)
  • 순수함수(Pure Function)
  • 고차 함수(High-Order Function)
  • 합성 함수(Function Composition)
  • 선언형(추상화가 높아지고 직관적이 된다. 사람이 이해하기 편리.)

→ 불변성과 순수함수를 지키고 최대한 선언형 방식의 프로그램이 되도록 지켜야할 규칙을 따르는 것

리액트는 함수 컴포넌트에 무게를 두고 있다

  1. 리액트의 핵심 개념 중 하나인 Hooks는 함수 컴포넌트에서만 동작

  2. 훅을 도입한 이유

    Classes confuse both people and machinesIn addition to making code reuse and code organization more difficult, we’ve found that classes can be a large barrier to learning React... However, we found that class components can encourage unintentional patterns that make these optimizations fall back to a slower path. Classes present issues for today’s tools, too.

    클래스는 사람과 기계를 혼란스럽게 합니다. 코드 재사용 및 코드 조직을 더 어렵게 만드는 것 외에도, 우리는 클래스가 React를 배우는 데 큰 장벽이 될 수 있다는 것을 발견했습니다... 그러나 우리는 클래스 컴포넌트가 이러한 최적화가 더 느린 경로로 되돌아가게 만드는 의도하지 않은 패턴을 장려할 수 있다는 것을 발견했습니다. 클래스는 오늘날의 도구에도 문제를 제기합니다.

  3. 리액트는 단방향 데이터 흐름을 강제하는 구조, 이는 함수형 접근에 더 잘 들어맞는 방식

    → 리액트의 상태 관리 라이브러리는 기본적으로 불변 상태를 받아 불변 상태를 반환하는 함수를 바탕으로 한다

클래스형 컴포넌트의 경우 중복 코드양이 많고 가독성이 떨어져 이를 해결하기 위해 등장
